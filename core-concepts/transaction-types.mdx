The `TransactionType` enum provides type-safe constants for filtering transactions by their operation type. This enum matches the transaction types available in the Hedera network.

## TransactionType enum

The complete `TransactionType` enum from the source code:


## Import and usage

```typescript
import { TransactionType } from '@hashgraph/hedera-mirror-node-ts';

// Use with transactions query
const txns = await transactions(mirrorClient)
  .setTransactionType(TransactionType.CRYPTOTRANSFER)
  .get();
```

## Transaction type categories

### Consensus Service (HCS)

Transactions related to the Hedera Consensus Service for topic-based messaging:

<AccordionGroup>
  <Accordion title="CONSENSUSCREATETOPIC">
    Creates a new consensus topic for publishing messages.
    
    ```typescript
    const topicCreations = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CONSENSUSCREATETOPIC)
      .setLimit(10)
      .get();
    ```
  </Accordion>
  
  <Accordion title="CONSENSUSDELETETOPIC">
    Deletes an existing consensus topic.
    
    ```typescript
    const topicDeletions = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CONSENSUSDELETETOPIC)
      .get();
    ```
  </Accordion>
  
  <Accordion title="CONSENSUSSUBMITMESSAGE">
    Submits a message to a consensus topic. Most commonly used HCS transaction type.
    
    ```typescript
    const messages = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CONSENSUSSUBMITMESSAGE)
      .setLimit(100)
      .order('desc')
      .get();
      
    console.log(`Found ${messages.transactions.length} topic messages`);
    ```
  </Accordion>
  
  <Accordion title="CONSENSUSUPDATETOPIC">
    Updates the properties of an existing consensus topic.
    
    ```typescript
    const topicUpdates = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CONSENSUSUPDATETOPIC)
      .get();
    ```
  </Accordion>
</AccordionGroup>

### Cryptocurrency

Transactions for account and HBAR operations:

<AccordionGroup>
  <Accordion title="CRYPTOCREATEACCOUNT">
    Creates a new Hedera account.
    
    ```typescript
    const accountCreations = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CRYPTOCREATEACCOUNT)
      .setResult('success')
      .get();
    ```
  </Accordion>
  
  <Accordion title="CRYPTOTRANSFER">
    Transfers HBAR or tokens between accounts. Most commonly used transaction type.
    
    Real example from the test suite:
    
    ```typescript
    const txnCursor = transactions(mirrorClient)
      .setAccountId('0.0.1234')
      .setLimit(10)
      .setType('credit')
      .setTransactionType(TransactionType.CRYPTOTRANSFER)
      .setResult('success');
      
    const txns = await txnCursor.get();
    
    if (txns.transactions.length > 0) {
      const creditTransfer = txns.transactions[0].transfers.find(
        txn => txn.account === '0.0.1234'
      );
      console.log(`Credit amount: ${creditTransfer?.amount}`);
      console.log(`Transaction name: ${txns.transactions[0].name}`);
    }
    ```
  </Accordion>
  
  <Accordion title="CRYPTODELETE">
    Deletes a Hedera account and transfers remaining balance.
    
    ```typescript
    const accountDeletions = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CRYPTODELETE)
      .get();
    ```
  </Accordion>
  
  <Accordion title="CRYPTOUPDATEACCOUNT">
    Updates properties of an existing account.
    
    ```typescript
    const accountUpdates = await transactions(mirrorClient)
      .setTransactionType(TransactionType.CRYPTOUPDATEACCOUNT)
      .get();
    ```
  </Accordion>
  
  <Accordion title="CRYPTOADDLIVEHASH">
    Adds a live hash to an account (rarely used).
    
    ```typescript

      .setTransactionType(TransactionType.CRYPTOADDLIVEHASH)
    ```

  </Accordion>
  
  <Accordion title="CRYPTODELETELIVEHASH">


## Combining with other filters

Transaction type filtering can be combined with other filters for powerful queries:

```typescript
import { 
  Client,
  transactions,
  TransactionType,
  optionalFilters 
} from '@hashgraph/hedera-mirror-node-ts';

const mirrorClient = new Client('https://testnet.mirrornode.hedera.com/');

// Get successful token mints for the last 24 hours
const oneDayAgo = `${Math.floor((Date.now() - 86400000) / 1000)}.000000000`;

const recentMints = await transactions(mirrorClient)
  .setTransactionType(TransactionType.TOKENMINT)
  .setResult('success')
  .timestamp(optionalFilters.greaterThan(oneDayAgo))
  .setLimit(100)
  .order('desc')
  .get();

console.log(`Found ${recentMints.transactions.length} token mints in the last 24 hours`);
```

## Response structure

The transaction response includes the transaction name matching the enum value:

```typescript
interface Transaction {
  bytes: null;
  charged_tx_fee: number;
  consensus_timestamp: string;
  entity_id: string;
  max_fee: string;
  memo_base64: string;
  name: string;  // Matches TransactionType enum value (e.g., "CRYPTOTRANSFER")
  nft_transfers: any[];
  node: string;
  nonce: number;
  parent_consensus_timestamp: null;
  result: string;  // e.g., "SUCCESS"
  scheduled: boolean;
  staking_reward_transfers: any[];
  token_transfers: any[];
  transaction_hash: string;
  transaction_id: string;
  transfers: Transfer[];
  valid_duration_seconds: string;
  valid_start_timestamp: string;
}
```

## Real-world usage patterns

### Pattern 1: Audit account activity

```typescript
const accountId = '0.0.1234';

// Get all transaction types for an account
const allTxns = await transactions(mirrorClient)
  .setAccountId(accountId)
  .setLimit(100)
  .get();

// Group by transaction type
const txnsByType = allTxns.transactions.reduce((acc, txn) => {
  acc[txn.name] = (acc[txn.name] || 0) + 1;
  return acc;
}, {} as Record<string, number>);

console.log('Transaction summary:', txnsByType);
```

### Pattern 2: Monitor token lifecycle

```typescript
// Track all operations for a token
const tokenId = '0.0.5678';

const tokenOperations = [
  TransactionType.TOKENCREATION,
  TransactionType.TOKENMINT,
  TransactionType.TOKENBURN,
  TransactionType.TOKENUPDATE,
  TransactionType.TOKENDELETION
];

for (const txnType of tokenOperations) {
  const txns = await transactions(mirrorClient)
    .setTransactionType(txnType)
    .setLimit(10)
    .get();
    
  console.log(`${txnType}: ${txns.transactions.length} transactions`);
}
```

### Pattern 3: HCS message analysis

```typescript
// Get all consensus submit message transactions
const messageTxns = await transactions(mirrorClient)
  .setTransactionType(TransactionType.CONSENSUSSUBMITMESSAGE)
  .setLimit(100)
  .order('desc')
  .get();

console.log(`Total messages: ${messageTxns.transactions.length}`);

// Analyze success rate
const successful = messageTxns.transactions.filter(
  txn => txn.result === 'SUCCESS'
).length;

const successRate = (successful / messageTxns.transactions.length) * 100;
console.log(`Success rate: ${successRate.toFixed(2)}%`);
```

## Type safety benefits

Using the enum provides type safety and IDE autocomplete:

```typescript
// ✓ Type-safe - IDE autocomplete available
await transactions(mirrorClient)
  .setTransactionType(TransactionType.CRYPTOTRANSFER)
  .get();

// ✗ Typo would cause compile error
await transactions(mirrorClient)
  .setTransactionType(TransactionType.CRYPTOTRANFER)  // Error!
  .get();
```
