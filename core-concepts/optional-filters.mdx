The SDK provides an `optionalFilters` helper object that creates comparison operators for filtering query results. These filters work with any parameter that accepts `OptionalFilters` type.

## The OptionalFilters type

The `OptionalFilters` type is a union type that represents comparison operators:

```typescript
export type OptionalFilters =
  | `gt:${string}`
  | `gte:${string}`
  | `lt:${string}`
  | `lte:${string}`
  | `ne:${string}`
  | `eq:${string}`
  | string;
```

This type allows you to pass either:
- A comparison operator string (e.g., `"gt:1000"`, `"lte:5000"`)
- A plain string value (direct equality)
- A value generated by the `optionalFilters` helper

## The optionalFilters helper

The `optionalFilters` object provides methods to create properly formatted comparison operators:

```typescript
export const optionalFilters = {
  greaterThan(val: any): OptionalFilters {
    return `gt:${val}`;
  },
  greaterThanEqualTo(val: any): OptionalFilters {
    return `gte:${val}`;
  },
  lessThan(val: any): OptionalFilters {
    return `lt:${val}`;
  },
  lessThanEqualTo(val: any): OptionalFilters {
    return `lte:${val}`;
  },
  notEqualTo(val: any): OptionalFilters {
    return `ne:${val}`;
  },
  equalTo(val: any): OptionalFilters {
    return `eq:${val}`;
  }
}
```

## Using with account queries

### Filter by account balance

Find accounts with balance greater than a threshold:

```typescript
import { Client, accounts, optionalFilters } from '@tikz/hedera-mirror-node-ts-ts';

const mirrorClient = new Client('https://testnet.mirrornode.hedera.com/');

// Get accounts with balance > 1000 HBAR (in tinybars)
const wealthyAccounts = await accounts(mirrorClient)
  .setBalance(optionalFilters.greaterThan(100000000000))  // 1000 HBAR in tinybars
  .setLimit(10)
  .get();

console.log(`Found ${wealthyAccounts.accounts.length} wealthy accounts`);
```

### Filter by account ID

Find accounts with ID greater than a specific value:

```typescript
// Get accounts created after account 0.0.1000
const recentAccounts = await accounts(mirrorClient)
  .setAccountId(optionalFilters.greaterThan('0.0.1000'))
  .setLimit(50)
  .get();
```

## Using with topic messages

Real-world example from the test suite showing sequence number filtering:

```typescript
import { 
  Client, 
  topicMessages, 
  optionalFilters 
} from '@tikz/hedera-mirror-node-ts-ts';

const mirrorClient = new Client('https://testnet.mirrornode.hedera.com/');

const limit = 10;
const seqNo = 20;

// Get messages with sequence number > 20
const msgCursor = topicMessages(mirrorClient)
  .setTopicId('0.0.1234')
  .setLimit(limit)
  .order('asc')
  .setSequenceNumber(optionalFilters.greaterThan(seqNo));

const msgs = await msgCursor.get();

// Verify all messages have sequence number > 20
if (msgs.messages.length === limit) {
  const lastSeq = msgs.messages[limit - 1].sequence_number;
  console.log(`Last message sequence: ${lastSeq} (should be > ${seqNo})`);
}
```

## Using with timestamp filters

All paginated queries support timestamp filtering:

```typescript
import { optionalFilters } from '@tikz/hedera-mirror-node-ts-ts';

// Get transactions after a specific timestamp
const txnCursor = transactions(mirrorClient)
  .setAccountId('0.0.1234')
  .timestamp(optionalFilters.greaterThan('1640000000.000000000'))
  .setLimit(100);

const txns = await txnCursor.get();
```

### Timestamp format

Hedera timestamps are in the format: `seconds.nanoseconds`

Example: `1640000000.123456789`

```typescript
// Get transactions in a time range
const startTime = '1640000000.000000000';
const endTime = '1640086400.000000000';

const txnsAfterStart = await transactions(mirrorClient)
  .timestamp(optionalFilters.greaterThanEqualTo(startTime))
  .get();

const txnsBeforeEnd = await transactions(mirrorClient)
  .timestamp(optionalFilters.lessThanEqualTo(endTime))
  .get();
```

## Combining filters

You can chain multiple filters together:

```typescript
import { 
  Client, 
  accounts, 
  transactions,
  optionalFilters,
  TransactionType 
} from '@tikz/hedera-mirror-node-ts-ts';

const mirrorClient = new Client('https://testnet.mirrornode.hedera.com/');

// Combine account ID, transaction type, and result filters
const txnCursor = transactions(mirrorClient)
  .setAccountId('0.0.1234')
  .setLimit(10)
  .setType('credit')
  .setTransactionType(TransactionType.CRYPTOTRANSFER)
  .setResult('success')
  .timestamp(optionalFilters.greaterThan('1640000000.000000000'));

const txns = await txnCursor.get();

// Verify the filters worked
if (txns.transactions.length > 0) {
  const creditTransfer = txns.transactions[0].transfers.find(
    txn => txn.account === '0.0.1234'
  );
  console.log(`Credit amount: ${creditTransfer?.amount}`);
  console.log(`Transaction name: ${txns.transactions[0].name}`);
  console.log(`Result: ${txns.transactions[0].result}`);
}
```

## Real-world examples

### Example 1: Find high-value transactions

```typescript
import { 
  Client, 
  transactions, 
  TransactionType 
} from '@tikz/hedera-mirror-node-ts-ts';

const mirrorClient = new Client('https://mainnet.mirrornode.hedera.com/');

const highValueTxns = await transactions(mirrorClient)
  .setTransactionType(TransactionType.CRYPTOTRANSFER)
  .setLimit(100)
  .order('desc')
  .get();

// Filter high-value transfers in your code
const filtered = highValueTxns.transactions.filter(txn => {
  const totalTransferred = txn.transfers.reduce((sum, t) => 
    sum + Math.abs(t.amount), 0
  );
  return totalTransferred > 100000000000; // > 1000 HBAR
});

console.log(`Found ${filtered.length} high-value transactions`);
```

### Example 2: Monitor account activity

```typescript
import { 
  Client, 
  transactions,
  optionalFilters 
} from '@tikz/hedera-mirror-node-ts-ts';

const mirrorClient = new Client('https://testnet.mirrornode.hedera.com/');

// Get recent activity for an account
const accountId = '0.0.1234';
const oneDayAgo = Date.now() - 86400000; // 24 hours ago
const timestamp = `${Math.floor(oneDayAgo / 1000)}.000000000`;

const recentActivity = await transactions(mirrorClient)
  .setAccountId(accountId)
  .timestamp(optionalFilters.greaterThan(timestamp))
  .setLimit(100)
  .order('desc')
  .get();

console.log(`${accountId} had ${recentActivity.transactions.length} transactions in the last 24 hours`);
```

### Example 3: Token operations analysis

```typescript
import { 
  Client, 
  transactions,
  TransactionType

const mirrorClient = new Client('https://mainnet.mirrornode.hedera.com/');
```

## Filter parameters reference

Different query types support different filterable parameters:

### Accounts

- `setAccountId(OptionalFilters)` - Filter by account ID
- `setBalance(OptionalFilters)` - Filter by account balance
- `timestamp(OptionalFilters)` - Filter by timestamp

### Transactions  

- `setAccountId(OptionalFilters)` - Filter by account ID
- `timestamp(OptionalFilters)` - Filter by timestamp

### Topic Messages

- `setSequenceNumber(OptionalFilters)` - Filter by sequence number
- `timestamp(OptionalFilters)` - Filter by timestamp

## Type safety

The `OptionalFilters` type provides type safety while allowing flexible usage:

```typescript
// All of these are valid:
accounts(client).setAccountId('0.0.1234')                        // Direct string
accounts(client).setAccountId('gt:0.0.1000')                     // Comparison string
accounts(client).setAccountId(optionalFilters.greaterThan('0.0.1000'))  // Helper

// Type checking prevents errors:
accounts(client).setBalance(optionalFilters.lessThan(1000))      // ✓ Valid
accounts(client).setBalance(1000)                                 // ✓ Valid (coerced to string)
```

## Implementation details

The `optionalFilters` helpers are simple string template functions:

```typescript
// Each function returns a formatted string
greaterThan(val: any): OptionalFilters {
  return `gt:${val}`;
}
```

These strings are then passed as query parameters to the Mirror Node REST API:

```
GET /api/v1/accounts?account.id=gt:0.0.1000&limit=10
GET /api/v1/transactions?timestamp=gte:1640000000.000000000
GET /api/v1/topics/{topicId}/messages?sequencenumber=gt:20
```
