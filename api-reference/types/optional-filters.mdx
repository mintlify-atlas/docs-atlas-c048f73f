## Overview

The `OptionalFilters` type and `optionalFilters` helper object provide a type-safe way to apply comparison operators to Mirror Node API queries. These filters allow you to specify conditions like greater than, less than, equality, and more.

## Type definition

```typescript
type OptionalFilters =
  | `gt:${string}`
  | `gte:${string}`
  | `lt:${string}`
  | `lte:${string}`
  | `ne:${string}`
  | `eq:${string}`
  | string;
```

The `OptionalFilters` type accepts:
- Prefixed strings with comparison operators (`gt:`, `gte:`, `lt:`, `lte:`, `ne:`, `eq:`)
- Plain strings for exact matching

## Helper methods

The `optionalFilters` object provides convenient methods to construct filter strings:

<ParamField path="greaterThan" type="(val: any) => OptionalFilters">
  Creates a greater-than filter. Returns values strictly greater than the specified value.
  
  ```typescript
  optionalFilters.greaterThan('0.0.1000')
  // Returns: "gt:0.0.1000"
  ```
</ParamField>

<ParamField path="greaterThanEqualTo" type="(val: any) => OptionalFilters">
  Creates a greater-than-or-equal-to filter. Returns values greater than or equal to the specified value.
  
  ```typescript
  optionalFilters.greaterThanEqualTo(1000000000)
  // Returns: "gte:1000000000"
  ```
</ParamField>

<ParamField path="lessThan" type="(val: any) => OptionalFilters">
  Creates a less-than filter. Returns values strictly less than the specified value.
  
  ```typescript
  optionalFilters.lessThan('1234567890.000000000')
  // Returns: "lt:1234567890.000000000"
  ```
</ParamField>

<ParamField path="lessThanEqualTo" type="(val: any) => OptionalFilters">
  Creates a less-than-or-equal-to filter. Returns values less than or equal to the specified value.
  
  ```typescript
  optionalFilters.lessThanEqualTo(500)
  // Returns: "lte:500"
  ```
</ParamField>

<ParamField path="notEqualTo" type="(val: any) => OptionalFilters">
  Creates a not-equal-to filter. Excludes values equal to the specified value.
  
  ```typescript
  optionalFilters.notEqualTo('0.0.98')
  // Returns: "ne:0.0.98"
  ```
</ParamField>

<ParamField path="equalTo" type="(val: any) => OptionalFilters">
  Creates an equality filter. Returns only values equal to the specified value.
  
  ```typescript
  optionalFilters.equalTo('SUCCESS')
  // Returns: "eq:SUCCESS"
  ```
</ParamField>

## Usage examples

### Filter transactions by timestamp

```typescript
import { Client, transactions, optionalFilters } from '@tikz/hedera-mirror-node-ts';

const client = new Client('https://testnet.mirrornode.hedera.com');

// Get transactions after a specific timestamp
const recentTxs = await transactions(client)
  .timestamp(optionalFilters.greaterThan('1234567890.000000000'))
  .get();
```

### Filter accounts by balance

```typescript
import { accounts, optionalFilters } from '@tikz/hedera-mirror-node-ts';

// Get accounts with balance greater than 100 HBAR
const richAccounts = await accounts(client)
  .setBalance(optionalFilters.greaterThan(10000000000)) // 100 HBAR in tinybars
  .get();

// Get accounts with specific balance range
const balanceRange = await accounts(client)
  .setBalance(optionalFilters.lessThan(50000000000)) // Less than 500 HBAR
  .get();
```

### Filter accounts by ID

```typescript
// Get accounts with ID less than a specific value
const accountsBelow = await accounts(client)
  .setAccountId(optionalFilters.lessThan('0.0.1000'))
  .setLimit(10)
  .get();

// Get accounts excluding a specific ID
const excludeAccount = await accounts(client)
  .setAccountId(optionalFilters.notEqualTo('0.0.98'))
  .get();
```

### Filter topic messages by sequence number

```typescript
import { topicMessages, optionalFilters } from '@tikz/hedera-mirror-node-ts';

// Get messages with sequence number greater than 20
const messages = await topicMessages(client)
  .setTopicId('0.0.16430')
  .setSequenceNumber(optionalFilters.greaterThan(20))
  .setLimit(10)
  .get();
```

## Notes

- All helper methods accept any value type and convert it to a string in the filter
- Multiple filters can be applied to the same parameter by passing an array
- The plain string value (without prefix) performs exact matching
- Timestamp values should be in the format `seconds.nanoseconds` (e.g., `1234567890.000000000`)
- Balance and supply values are typically in the smallest unit (tinybars for HBAR, base units for tokens)
